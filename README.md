## 一、项目概述

本项目的核心目标是构建一个优雅、快速、易部署的个人/团队书签收藏与分享平台。

### 核心理念
- ✅ 极简部署 - Cloudflare 零服务器运维
- ✅ 极致性能 - 全球边缘节点秒级加载
- ✅ 数据自主 - D1+KV 个人云端存储

---

## 二、第一性原理分析

问题：为何做导航站？   解决方案：集中管理分散的常用网站链接，减少记忆成本

问题：为何选 Cloudflare？解决方案：免费额度 + 全球边缘节点 + 无需传统服务器

问题：数据存哪里？     解决方案：D1(关系型数据库)+ KV(键值对存储)

问题：谁可访问？       解决方案：公开前台展示 + 私域后台管理

---

## 三、系统架构

浏览器 <---> Cloudflare Worker(逻辑) <---> Cloudflare D1 Database(书签数据)
   ^                 ^                        ^
HTML/Tailwind      无状态执行              结构化查询
CDN缓存中          边缘计算                KV认证

---

## 四、技术栈清单

Cloudflare Workers | 后端逻辑 | 全球边缘运行
D1 | 关系型数据库 | 书签数据存储
KV | Key-Value 存储 | 会话认证凭据
Tailwind CSS | UI 样式 | 响应式主题定制
JavaScript | 编程语言 | 单文件脚本实现

---

## 五、核心功能

【前台功能】
📱 响应式设计 - 桌面/平板/手机全适配
🔍 模糊搜索 - 站内即时定位所需书签
📂 分类展示 - 分组浏览，结构清晰
⚡ 秒级加载 - 利用 CF 边缘缓存
📤 导入导出 - config.json 格式互通

【后台功能】
🔒 管理员认证 - KV 存储凭据，表单登录
🛡️ HttpOnly Cookie - 12小时有效会话，URL不暴露密码
🗂️ 分类排序 - 自定义分类显示顺序
✏️ CRUD操作 - 增删改查书签
🔑 退出登录 - 一键销毁会话

## 六、部署流程（7步）

步骤1: 账号准备 - 注册 Cloudflare 账号 - 5分钟
步骤2: 创建D1 - 建立 book 数据库 - 2分钟
步骤3: 建表指令 - 执行 SQL 初始化 - 3分钟
步骤4: 创建KV - 命名空间 NAV_AUTH - 1分钟
步骤5: 设置凭证 - admin_username & password - 1分钟
步骤6: 部署Worker - 绑定服务 - 5分钟
步骤7: 上传代码 - 粘贴 work_v2.js 部署 - 2分钟

总计：约 19分钟 完成从0到可用

## 七、V2 版本关键升级

V1版本 ❌    V2版本 ✅
URL参数传递账号密码 -> 表单+HttpOnly Cookie
无有效期限制 -> 12小时自动过期
无退出按钮 -> 右上角一键注销
默认字母序 -> 自定义 order 数值
默认开放投稿 -> 环境变量控制开关
无XSS防护 -> URL转义+HTML过滤双层防护

---

## 八、安全性矩阵

风险等级：高风险 | 威胁面：凭据泄露 | 缓解措施：URL传参已废弃 -> KV加密存储

风险等级：中风险 | 威胁面：XSS注入 | 缓解措施：HTML转义+输入校验双重防护

风险等级：低风险 | 威胁面：未授权访问 | 缓解措施：后台路径 /admin 需登录凭证

风险等级：低风险 | 威胁面：爬虫抓取 | 缓解措施：Cloudflare 自带WAF保护

风险等级：低风险 | 威胁面：数据篡改 | 缓解措施：Admin权限隔离，普通访客只读

## 九、定制化开发指南

修改主题颜色:
在 work_v2.js 头部配置 tailwind.config 中的 primary 颜色值

关闭访客投稿:
在 Worker 环境变量中添加 ENABLE_PUBLIC_SUBMISSION=false

自定义排序规则:
书签：后台"排序"值越小越靠前
分类：写入 category_orders 表或后台标签页编辑

## 十、版本兼容性

v1 → v2: 替换为 work_v2.js (旧URL登录失效，需重新登录)
<v1 → v1: ALTER TABLE sites 加字段 (sort_order 新字段必加)
跨大版本: 先备份config.json (支持导入/导出无缝迁移)
